Git

Git is a free and open source distributed version control system designed to handle everything from small to very large projects with speed and efficiency. Git is software for tracking changes in any set of files, usually used for coordinating work among programmers collaboratively developing source code during software development. Its goals include speed, data integrity, and support for distributed, non-linear workflows (thousands of parallel branches running on different systems).

Getting Started with Git

  • Follow this tutorial step by step:
  • First of all you need to download Git (for Windows/Linux/MacOS). Click Here to Download
  • Once you have download and Install it, head over to the Command Prompt to make sure it's there. Type git --version in Comand Prompt

Creating a Repository

Once you have everything setup and have Git access to your local machine. This is the time to Crate a Repository.

  • Create a Folder you want to Intilize as a Repostiry.
  • Open Comand Prompt and Navigate to that Folder, you just created.
  • Now in the terminal type "git init" this will initialize your folder as a GitHub Repository.

Adding Changes

After you have intialized a repository, you can create some files make some chnages to them.

  • After making some changes. Type "git add ." to register all your changes to branching.

Checking Status

After you have added your changes to branching. You can check status of your files that are changes and ready to commit.

Type "git status" and it shows all currently modiefied files.

Making Commit

After yo have intialized a repository, you can create some files and commit all these chnages to your repository.

  • First of all create a File and write some lines in it.
  • After making some changes. Type "git add ." to register all your changes to branching.
  • Then type "git commit -m <'message'>" to commit all your changs.

Checking Recent Commits

After you have made some chnages and commited them. You can view information about recent commits i.e; Who made the commit, their email and Date time of the commit.

  • Type "git log" and it shows all information.

Github

If you are working with a team on a same project and all of them need access to up-to-date files at all times, Githb is something that can solve your problems. GitHub is a cloud storage platform. Here you can keep all your neccesary files as repositories and you an your peers can work on them simultaneously and have acces to all the updated files at all times, anywhere.

Creating GitHub Repository

  • To create a GitHub repository, you must have a github account and you must be logged in.
  • After logging in to your GitHub account, on the top write corner next to your profile icon, click the + sign and select "New Repository"

  • Now give your repository a name, initialize it with a README (just so it isn't empty) and click "Create Repository".

Linking GitHub and Git Repositories

After you have created your GitHub repository and also a Git repository on your local machine. You can now link them both and keep all your changes on local and remote repository, synchronized.

Follow these Steps to link your repositories.

  • Copy your Remote repository link.
  • Now head over to the termianl, and navigate the folder, you intialized as a local repository.
  • Type "git remote add origin <'your repo linl'> origin main"

Pushing Commits to Remote Repository

As you have your local repository set up and also have some commits on it. Now you can push all your local commits to your remote repository.

Follow these Steps:

  • Type "git push -u origin master". This will push all your local commits to remote repository.
  • Now if you see your remote repository. It should have all your files.

Wrapping it up

That's all about basics of Git and GitHub the rest is simple if you have grip on the basics.

Now everytime you make changes to your local repo. You Follow this sequence.

  • git add .
  • git commit -m "message"
  • git push -u origin main

Doing this step before everytime you are about to make crutial changes to your files, is recommended. So that you can roll-back to your privious version, if an unfortunate event occurs.
